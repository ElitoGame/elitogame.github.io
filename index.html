DnDBot

Quick Profile:

Type: Discord Bot

Language(s): JavaScript, Bash, SQL

APIs/ORMs: Discord API, Sequilize ORM, Giphy SDK

This project was made for me and a couple of friends who wanted to play DnD.
Due to the current situation we couldn't meet in real live, so we used the platform Discord. In order to support us, I coded a Bot using the Discord API which would allow us to create and edit a Character sheet as well as a set of dices, accessible for everyone of us as long as the Bot was running.

The Bot is running on my Laptop started by a Batch Script whenever I turn my PC on. I wrote the Bot using JavaScript, the Discord API and NodeJS in order to host it.
Some of the Bots features required me to store data, so I used the Database SQLite and accessed it with the ORM Sequilize. For a joke, I also implemented the Giphy SDK which allowed me to display a random "shuffle" GIF upon the command "?shuffle".

Kineticraft

A Minecraft Plugin.

Kineticraft

Quick Profile:

Type: Java Minecraft Plugin

Language(s): Java, SQL

Implements: Multiple other plugins including Grief Prevention, CoreProtect, WorldGuard and more

This was my first big project and I am still working on it every so often. I wrote this plugin for the Kineticraft Minecraft server [kineticraft.net] and I am keeping it updated, modifying it depending on the need of the server.
Currently there are 14 commands, 19 custom mechanics and some additional features like NPCs with custom GUIs and customizable Items.

A bunch of those mechanics are adjustable using a config.yml and the NPC, GUI and Item feature even get their own configuration file which allow me to modify the gameplay on the fly without needing to restart the server.
The code for this plugin is kept on Github, but we decided on keeping it private in order to avoid possible exploits.
Of course this plugin also has to store a bunch of data and so I used .json files but switched a few months ago to a SQLite database using the java.sql package as it was way more organized.

GoDot & Electron

Quick Profile:

Type: GoDot Project | Electron App | Browser Game

Language(s): GDScript, JavaScript

I also worked on a bunch of other projects which only made it into the proof of concept stage. Here, I wanted to showcase some of my favourites:

  • NodeIt:
    The goal was to make a App allowing me to take notes and plan out ideas using a system of nodes.
    My first approach of using the GoDot Game Engine was kind of successful, but I struggled with a few of it's options, so I switched to Electron, a framework that enables me to create Desktop Apps using the tools I was used from the browser. I didn't finish this project either due to school based time limitations, but I was able to create a nice GUI, spawn in nodes, connect them, scale the board & move around. In my GoDot attempt, I was able to spawn in nodes, insert text, remove individual nodes & move around.
  • Marauders:
    I started the project Marauders as a Birthday present for someone who loves Harry Potter and the Marauders Map and so, I made two versions of it. The first one was made for the Browser using a lot of JS in order to calculate the footsteps position and movement.
    The second one was made using the GoDot Game Engine, which made creating a game far easier and also allowed me to use TileSets and the inbuilt collision Engine.
  • Simple 2D Platformer:
    I had some Experience in the GoDot Game Engine because I created a simple Platformer before. For this project, I coded a simple enemy which you could jump on, and coded in the basic movement for the character including a double jump and wall jump.
